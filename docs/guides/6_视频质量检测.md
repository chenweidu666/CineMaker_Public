# 视频质量检测（规划中）

AI 生成的视频常出现手部畸形、人物不一致、面部崩坏等问题。本功能通过 ffmpeg 切帧 + Vision 模型逐帧分析，自动检测质量问题，输出可用性判断和剪辑建议。

## 已有基础设施

| 能力 | 实现位置 | 说明 |
|------|----------|------|
| 任意时间戳切帧 | `pkg/video/scene_detect.go` → `extractFrame()` | ffmpeg `-ss` 精确提取单帧 |
| 视频时长获取 | `ffmpeg.GetVideoDuration()` | 用于计算切帧间隔 |
| Vision 多模态分析 | `pkg/ai/openai_client.go` → `GenerateTextWithImages()` | 支持图片+文本输入 |
| 角色参考图 | `characters` 表 `reference_image` 字段 | 用于人物一致性对比 |
| 视频本地路径 | `VideoGeneration.LocalPath` | 已有完整的本地存储和 COS 链接 |

## 整体架构

```
输入：storyboard_id / episode_id（批量）
      │
      ▼
┌─────────────────┐
│  1. 查询视频记录  │  从 video_generations 表获取视频路径
└──────┬──────────┘
       │
       ▼
┌─────────────────┐
│  2. ffmpeg 切帧  │  每 0.5 秒提取一帧（PV 通常 5-8 秒，约 10-16 帧）
└──────┬──────────┘
       │
       ▼
┌─────────────────┐
│  3. Vision 分析  │  每帧 + 角色参考图 → Vision 模型（并发上限 3）
└──────┬──────────┘
       │
       ▼
┌─────────────────┐
│  4. 汇总报告     │  逐帧评分 + 问题标注 + 整体可用性 + 剪辑建议
└─────────────────┘
```

## 检测项

| 类别 | 检测内容 | 严重程度 |
|------|----------|----------|
| 手部畸形 | 多余手指、断指、关节反转、手部融合 | 高 |
| 肢体异常 | 多余肢体、肢体断裂、比例失调 | 高 |
| 面部崩坏 | 五官扭曲、不对称、表情不自然 | 高 |
| 人物一致性 | 面部/服装/发型与参考图不符 | 中 |
| 画面质量 | 模糊、伪影、闪烁痕迹、色块 | 低 |

## Vision Prompt 设计

每帧发送：**当前帧截图** + **角色参考图（三视图）**

```
你是 AI 生成视频的质量检测专家。请分析这张视频帧截图，对比角色参考图，逐项检查：

1. 手部：手指数量是否正确（每只手5根），有无断指、融合、关节反转
2. 肢体：肢体数量是否正确（两臂两腿），有无断裂、多余、比例失调
3. 面部：五官是否正常、对称、表情自然
4. 人物一致性：面部特征、服装颜色款式、发型是否与参考图一致
5. 画面质量：是否有模糊区域、伪影色块、不自然的纹理

返回 JSON：
{
  "score": 1-10,
  "issues": [
    {"type": "hand|body|face|consistency|artifact", "severity": "low|medium|high", "description": "具体问题描述"}
  ],
  "usable": true/false
}

评分标准：
- 9-10：无明显问题
- 7-8：有轻微瑕疵但不影响观看
- 5-6：有明显问题，需要评估是否可用
- 1-4：严重问题，建议重新生成
```

## API 设计

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/videos/:id/quality-check` | 对单个视频发起质检 |
| POST | `/api/v1/episodes/:id/quality-check` | 批量质检一集所有视频 |
| GET | `/api/v1/videos/:id/quality-report` | 获取质检报告 |

## 切帧策略

- PV 视频（5-8 秒）：每 0.5 秒一帧，首尾帧额外提取
- 普通剧集视频（4-12 秒）：每 1 秒一帧
- 帧图片保存到 `frames/quality/sb{id}/` 目录，质检完成后可选清理

## 工作量估算

- 后端服务 + API：约 300-400 行 Go 代码
- 前端质检按钮 + 报告展示：约 100 行 Vue 代码
- Vision prompt 调优：迭代 2-3 轮
