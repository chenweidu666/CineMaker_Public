# 视频分析流水线

CineMaker 的视频分析功能可以自动解析一段视频，提取镜头、语音和画面信息，最终由 LLM 合成为结构化剧本。整个流程分为五个阶段，前端以时间线形式实时展示每一步的进度与结果。

## 整体架构

```
用户输入 (URL / 本地上传)
      │
      ▼
┌─────────────┐
│  1. 下载视频  │  yt-dlp / 本地文件
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  2. 场景切分  │  FFprobe 帧差检测 + FFmpeg 关键帧提取 + 音频分离
└──────┬──────┘
       │
       ├─────────────────┐
       ▼                 ▼
┌─────────────┐   ┌─────────────┐
│ 3. 语音识别  │   │ 4. 画面分析  │  ← 并行执行
│  (BigASR)   │   │(Vision LLM) │
└──────┬──────┘   └──────┬──────┘
       │                 │
       └────────┬────────┘
                ▼
       ┌─────────────┐
       │  5. 剧本合成  │  LLM 综合输出 JSON 剧本
       └─────────────┘
```

## 阶段详解

### 1. 下载视频

| 项目 | 说明 |
|------|------|
| 实现位置 | `application/services/video_analysis_service.go` → `SubmitURL` / `SubmitUpload` |
| 工具 | [yt-dlp](https://github.com/yt-dlp/yt-dlp)（URL 下载）或直接接收上传文件 |
| 输出 | 本地 MP4 文件路径、视频标题 |
| 阶段数据 | `stage_data.download = { status, title, duration }` |

支持小红书、抖音、B 站等平台链接。前端提示用户可通过 [hellotik.app](https://hellotik.app) 手动下载后上传。

### 2. 场景切分

| 项目 | 说明 |
|------|------|
| 实现位置 | `pkg/video/scene_detect.go` → `DetectScenes()` |
| 算法 | FFprobe `scene` 滤镜（帧间像素差值），阈值默认 0.3 |
| 回退策略 | 若检测结果为空，按固定间隔（2s）采样 |
| 输出 | `SceneDetectResult`：镜头列表（起止时间 + 关键帧图片路径）+ 音频文件路径 |
| 阶段数据 | `stage_data.detect = { shot_count, duration, has_audio, shots[] }` |

关键帧通过 FFmpeg 的 `-ss` 精确定位到每个镜头的起始时间点提取。音频使用 `-vn -acodec pcm_s16le` 提取为 WAV 格式。

### 3. 语音识别（并行）

| 项目 | 说明 |
|------|------|
| 实现位置 | `pkg/asr/volcengine_asr.go` → `TranscribeFile()` |
| 服务 | 火山引擎 BigASR 录音文件识别（异步提交 + 轮询） |
| 数据传输 | 本地文件 Base64 编码后通过 `audio.data` 字段提交 |
| 输出 | `[]TranscriptSegment`：带时间戳的文本片段 |
| 阶段数据 | `stage_data.transcribe = { status, message, segments[] }` |
| 进度回调 | 每次轮询查询时通过 `OnProgress` 回调更新前端 |

ASR 采用异步模式：先 `submit` 提交任务，再以 3-10 秒间隔 `query` 轮询结果。若音频为静音（状态码 `20000003`），不视为致命错误。

### 4. 画面分析（并行）

| 项目 | 说明 |
|------|------|
| 实现位置 | `application/services/video_analysis_service.go` → `analyzeFramesLive()` |
| 模型 | 火山引擎豆包 Vision 模型（`doubao-seed-1-6-vision-250815`） |
| API | `/api/v3/responses`（Volcengine Responses API） |
| 并发控制 | 信号量限制最多 3 帧同时分析 |
| 输出 | 每帧的画面描述文本 |
| 阶段数据 | `stage_data.analyze = { status, done, total, frames[] }` |
| 实时更新 | 每完成一帧即刷新 stageData，前端可看到逐帧结果 |

每帧图片经 Base64 编码后作为 `input_image` 发送给 Vision 模型。提示词要求模型详细描述人物外貌、场景环境、动作、表情、光线等信息。

### 5. 剧本合成

| 项目 | 说明 |
|------|------|
| 实现位置 | `application/services/video_analysis_service.go` → `synthesizeScript()` |
| 模型 | 通用文本 LLM（通过 `ai_service_configs` 中 `service_type='text'` 配置） |
| 输入 | 镜头画面描述 + 语音转写文本 |
| 输出 | JSON 结构化剧本 |

LLM 从五个维度分析视频内容：

1. **角色设计** — 统一角色名称，详细外貌特征描述（发型、服装、气质等）
2. **道具设计** — 标注关键道具和物品
3. **场景设计** — 推断场景地点，注意场景连续性
4. **对话语言** — 分配对白给具体角色
5. **剧情变化** — 通过镜头顺序和情绪体现故事弧线

输出 JSON 结构：

```json
{
  "title": "剧本标题",
  "summary": "剧情概述",
  "tags": ["都市情感", "喜剧搞笑"],
  "characters": [
    { "name": "角色名", "description": "外貌描述", "role": "主角" }
  ],
  "shots": [
    {
      "index": 0,
      "start_time": 0.0,
      "end_time": 5.0,
      "description": "画面内容",
      "location": "场景地点",
      "characters": ["角色名"],
      "dialogue": "角色名：台词内容",
      "mood": "情感氛围"
    }
  ]
}
```

预设标签（LLM 从中选取 2-5 个）：都市情感、古装仙侠、悬疑推理、喜剧搞笑、科幻未来、恐怖惊悚、青春校园、职场商战、家庭伦理、动作冒险、文艺清新、纪录写实、美食探店、旅行Vlog、知识科普、开箱测评、穿搭时尚、美妆护肤、健身运动、宠物日常、游戏实况、音乐舞蹈、手工DIY、亲子育儿。

## 错误处理

- **单路失败中止**：语音识别或画面分析任一路出现致命错误时，跳过剧本合成阶段，任务标记为失败
- **静音容忍**：ASR 返回无识别结果（静音）不算致命错误，继续合成
- **全帧失败**：所有帧的 Vision 分析均失败才视为致命错误
- **失败任务管理**：前端为失败任务提供删除按钮，后端同时清理本地存储目录

## 实时进度机制

后端通过 `stage_data`（JSON）字段记录每个阶段的中间结果，前端以 2 秒间隔轮询 `/api/v1/video-analysis/:taskId` 获取最新状态。

| 机制 | 说明 |
|------|------|
| 画面分析逐帧推送 | 每完成一帧，`stage_data.analyze.frames[]` 追加结果，`done` 计数递增 |
| ASR 轮询进度 | 每次查询回调更新 `stage_data.transcribe.message` |
| 并行 stageData 保护 | 语音识别和画面分析共用 `stageData`，通过 `sync.Mutex` 保证并发安全 |
| 前端自动展开 | 当某阶段的 `stage_data` 出现后，对应面板自动展开（可手动折叠/展开） |

## 导入为剧本

分析完成后，用户可一键将结果导入为 CineMaker 剧本项目，自动创建：
- 短剧记录（标题、摘要、风格）
- 章节
- 角色
- 场景（按镜头中的 `location` 去重）
- 分镜（保留时间轴、对白、情绪等属性）

## 配置要求

在系统设置的 AI 服务配置页面中需要配置：

| 服务类型 | 用途 | 必需字段 |
|----------|------|----------|
| `text` | 文本 LLM（剧本合成） | API Key, 模型名, Base URL |
| `vision` | 视觉 LLM（画面分析） | API Key, 模型名, Base URL |
| `asr` | 语音识别 | App ID (`provider`), Access Token (`api_key`), Resource ID (`model_name`) |
